We respect the privacy of users of the "Mohasba Al-Nafs" browser extension. This page explains what data the extension collects, how it is used, why, and how you can control it.

1) Overview

"Mohasba Al-Nafs" is a browser extension that helps users track and follow up on their daily habits (self-accountability check-ins, dhikr, points, and reports), and opens automatically whenever a new tab is opened. The extension runs entirely locally on the user's device, with an optional Google Sign-In feature for syncing data across multiple devices.

2) Data We Collect

Data typePurpose
Authentication data Optional Google Sign-In via Firebase Authentication, used to sync user data across multiple devices
Basic personal identifiers (name and email) Received from the user's Google account only at sign-in, to link the account to stored data
Geographic location (GPS / Location) Used to accurately calculate prayer times via the external aladhan.com service
Local usage data (daily check-ins, points, statistics, settings) Stored locally on the user's device (Local Storage), and only synced if the user signs in
Note: Google Sign-In is entirely optional. If the user does not sign in, the extension works with full functionality locally on the device only, without sending any personal data to any external server — except for the prayer-times request sent to aladhan.com.

3) How We Use the Data

  • Saving the user's daily progress (check-ins, dhikr, points) to display it back to them later.
  • Syncing this data across the user's devices if they choose to sign in.
  • Calculating accurate prayer times based on the user's geographic location.
  • Scheduling daily reminders and notifications (morning/evening/dhikr) according to the user's settings.

4) Who We Share Data With

  • We do not sell, rent, or share any personal data with third parties for commercial or advertising purposes.
  • Firebase (Google) is used only to store and sync account data for users who choose to sign in.
  • aladhan.com is used only to fetch prayer times based on the geographic coordinates sent, and no personal identifiers are sent to it.

5) Permissions Used by the Extension

  • storage: to save the user's data locally on their device.
  • notifications: to send morning/evening reminders and dhikr reminders.
  • alarms: to schedule these reminders at specific times, even if the tab is closed.
  • identity: to enable optional Google Sign-In (Firebase Authentication).
  • Host permissions: to connect only to the Firebase and aladhan.com services.

6) Data Security

Usage data is stored locally on the user's device via the browser's Local Storage. Account data for users who choose to sync is stored through Google's Firebase infrastructure, which applies industry-standard security practices. We do not keep any additional copy of user data outside this platform.

7) User Rights

  • Users can use the extension fully without signing in, and therefore without sharing any personal identifiers.
  • Users can delete their local data at any time by clearing the extension's data from the browser settings.
  • Users can request deletion of their synced data (linked to their Google account) by contacting us via the email below.
